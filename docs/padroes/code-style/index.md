# Estilo de Código

## Convenções Gerais

- Use camelCase para variáveis e funções em JavaScript/TypeScript.
- Use PascalCase para classes e componentes.
- Indentação: 2 espaços.
- Linhas não devem exceder 80 caracteres.

## TypeScript Específico

- Sempre use tipos explícitos para parâmetros e retornos de função.
- Evite `any`; use tipos específicos.

## Flutter/Dart

- Use snake_case para nomes de arquivos.
- Seguir as convenções do Dart.

## Ferramentas

- ESLint para JavaScript/TypeScript
- Dartfmt para Dart
