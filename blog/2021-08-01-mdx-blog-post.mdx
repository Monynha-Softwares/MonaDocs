The Inclusive Design Lab is our new playground for testing UI components, motion guidelines, and accessibility guardrails before they land in production.

:::info Why this matters
70% of our active projects ship user-facing interfaces. Testing these experiences in isolation helps us deliver consistent, inclusive interactions across the board.
:::

<!-- truncate -->

## Whatâ€™s inside

- **Component sandboxes** powered by Storybook and Playroom. Designers and engineers can co-edit states in real time.
- **Automated audits** that run Axe, Pa11y, and color-contrast checks every time a sandbox changes.
- **Usage analytics** to reveal which variants are used across projects, helping us clean up duplicates.

## Getting hands-on

```jsx live
function MotionToggleExample() {
  const [reducedMotion, setReducedMotion] = React.useState(false);
  return (
    <div className={`demo ${reducedMotion ? 'demo--minimal-motion' : ''}`}>
      <p>Preview the card animation with motion controls:</p>
      <label className="toggle">
        <input
          type="checkbox"
          checked={reducedMotion}
          onChange={(event) => setReducedMotion(event.target.checked)}
        />
        <span>Enable reduced motion</span>
      </label>
      <div className="card">
        <h4>ArtLeo Exhibit Launch</h4>
        <p>Subtle hover and focus states respect user preferences.</p>
      </div>
    </div>
  );
}

render(<MotionToggleExample />);
```

## How to contribute

1. Add a Storybook story or Playroom example for the component you are proposing.
2. Document accessibility considerations in `docs/identity/ui-components.md` and cross-link from the project page.
3. Submit a PR and include screenshots or recordings; provide alt text for every visual asset.
